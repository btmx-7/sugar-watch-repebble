Why this wins

  • ✓Creativity — CGM on Pebble was beloved but broken. Reviving it meaningfully for PT2 is emotionally resonant for the T1D/T2D community
  • ✓Cleverness — Uses color zones, adaptive Quick View layout, touch alert dismiss, and persistent cold-start data — all new PT2 capabilities
  • ✓New platform use — Built primary for Emery (200×228, 64-color); secondary for Gabbro. Existing CGM watchfaces only target Basalt/Diorite
  • ✓Design quality — Medical-grade clarity. Color encodes urgency not decoration. Every pixel serves the glance interaction
  • ✓Hearts potential — The T1D community is large, vocal, and organized across Reddit, Facebook groups, Nightscout forums, and TuDiabetes

UX decisions

Glance-first
Color alone conveys zone. User knows if they're OK before reading the number.
Zero-noise baseline
No animation when in range. Alerts only fire for urgent conditions.
Persistent cold-start
Last known value shown immediately from persist storage. Never blank on wake.

Key functions

graph_layer_update_proc()
Adaptive Y-scale. Dashed threshold lines at low/high. Segments colored by zone. Dot at latest value.
start_alert_flash()
700ms flash timer. Cycles bg ↔ zone color. Triple-pulse haptic. Cancelled by SELECT or zone change.
prv_layout_for_bounds()
UnobstructedArea API. Compresses layout when Quick View active. Date hidden. Graph shrinks.

Publication checklist

  • Set up CloudPebble or local SDK project as "watchface"
  • Add Emery + Gabbro to targetPlatforms in package.json
  • Add a real BT icon PNG to resources/images/bt_icon.png
  • Build + test in emulator: pebble build && pebble install
  • Test Quick View: pebble emu-set-timeline-quick-view on
  • Publish: pebble publish — generates screenshots/GIFs automatically
  • Post in Nightscout Facebook groups, r/diabetes, r/pebble, TuDiabetes — with demo GIF
  • Put source on GitHub (public) — increases trust + hearts from developers
